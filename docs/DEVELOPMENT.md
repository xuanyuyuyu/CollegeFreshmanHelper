## 2. 后端模块说明

### `auth`

负责注册登录、登录态、当前用户信息。

### `forum`

负责帖子、回复、点赞、浏览统计，是当前最完整的业务模块。

### `user`

负责个人中心、资料与统计视图。

### `admin`

负责后台治理能力，包括：

- 管理员账号管理
- 帖子治理
- 评论治理
- 用户封禁
- 头衔管理
- 知识库管理
- 操作日志

### `common`

放置通用响应、异常、分页、配置等基础设施。

## 3. 当前后台管理能力

### 管理员账号

- 超级管理员新增普通管理员
- 超级管理员启用 / 禁用普通管理员
- 支持关键词和状态筛选

### 帖子管理

- 支持分页
- 支持按标题、正文、作者、帖子 ID 搜索
- 支持按状态、可见性筛选
- 支持隐藏、恢复、删除帖子

### 评论管理

- 支持分页
- 支持按评论内容、作者、被回复用户、来源帖子、帖子 ID 搜索
- 支持按状态、可见性筛选
- 支持隐藏、恢复、删除评论

### 用户治理

- 支持分页
- 支持按账号、昵称、用户 ID 搜索
- 支持角色、状态筛选
- 管理员可封禁普通用户
- 管理员可解封普通用户
- 普通管理员不能处理管理员账号

### 其他后台能力

- 头衔授予 / 撤销
- 知识库新增 / 上下线
- 操作日志查询

## 4. 核心设计约束

### 回复模型

- 数据层允许回复任意回复
- 展示层强制压成双层
- 避免无限楼中楼导致前端复杂度和查询膨胀

### 点赞模型

- Redis 记录用户点赞关系
- Redis 暂存最新点赞数
- 定时任务回刷 MySQL
- 避免高并发直接打数据库

### 浏览数模型

- 当前仅对登录用户去重
- Redis 键按 `postId + userId` 隔离
- TTL 12 小时

## 5. 本地联调建议

### 后端

```bash
mvn spring-boot:run
```

### 前端

```bash
cd frontend
npm install
npm run dev
```

### 编译检查

```bash
mvn -q -DskipTests compile
cd frontend && npm run build
```

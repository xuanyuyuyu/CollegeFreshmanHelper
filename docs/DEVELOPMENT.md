## 2. 模块说明

- 前端文案叫“账号”，后端字段仍是 `username`

- 通用响应

- 静态资源映射
- MyBatis-Plus 分页配置

## 3. 当前核心规则

### 3.1 回复模型

- 数据层允许回复任意回复
- 展示层强制压成双层
- 一级回复删除时级联删除子回复
- 二级回复删除时仅删自身

### 3.2 点赞模型

- 点赞关系持久化在 `forum_like_record`
- 点赞时间可持久化查询
- Redis 负责关系缓存和计数缓存
- 定时回刷最终计数到 MySQL

### 3.3 头衔模型

- 用户只能拥有一个头衔
- 用户也只能佩戴这一个头衔
- 管理员手动头衔优先级最高
- 自动头衔根据 `sys_title(grant_type=1)` 规则计算
- 自动头衔变更时直接更新用户当前唯一头衔记录

### 3.4 统计模型

`user_stats` 当前至少包含：

- 回复数
- 获赞数
- 知识贡献数
- 精选回答数

其中：

- `knowledgeContributionCount` 来源于 `knowledge_qa_trace.contributor_user_id`
- `featuredAnswerCount` 来源于 `knowledge_qa_trace.source_reply_id is not null`

## 4. 后台管理能力现状

- 超级管理员新增普通管理员
- 超级管理员启用 / 禁用普通管理员
- 支持筛选和列表查询

### 帖子管理

- 支持搜索
- 支持隐藏、恢复、删除帖子

### 评论管理

- 支持搜索
- 支持隐藏、恢复、删除评论

- 支持按账号 / 昵称 / 用户 ID 搜索

### 头衔管理

- 读取头衔列表
- 按账号授予头衔
- 撤销头衔
- 查看用户当前头衔

### 知识库管理

- 新增知识条目
- 上下线知识条目
- 查询知识条目
- 联动刷新贡献统计和自动头衔

## 5. 论坛前端注意事项

### 5.1 当前已做

- 分类筛选走后端真实筛选
- 无限滚动
- 详情返回列表时恢复滚动位置

### 5.2 后续改动注意

- 当前恢复逻辑基于 `sessionStorage`
- 当前恢复的是 `window.scrollY`
- 如果后面改成局部滚动容器，要同步改保存 / 恢复逻辑
- 如果要做“返回后静默刷新”，要避免旧请求覆盖新请求
